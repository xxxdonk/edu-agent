# 机器学习概述

## 学习目标
1. 理解机器学习的定义、核心思想及其与传统编程的区别
2. 掌握监督学习、无监督学习、强化学习三大范式的定义、典型任务与评价方式
3. 理解训练集、验证集、测试集的划分原则与作用
4. 掌握过拟合、欠拟合的概念及其与偏差-方差权衡的关系
5. 能够识别现实问题属于哪种学习类型并选择合适的基本方法

## 核心概念
- **机器学习 (Machine Learning)**：从数据中自动学习规律，并利用所学规律对未知数据进行预测或决策的方法论。Tom Mitchell 的经典定义：对于某类任务 T 和性能度量 P，若计算机程序在 T 上以 P 衡量的性能随经验 E 而改善，则称该程序从经验 E 中学习。
- **监督学习 (Supervised Learning)**：训练数据包含输入特征 X 和标签 Y，目标是学习从 X 到 Y 的映射函数。典型任务包括分类（预测离散类别）和回归（预测连续数值）。
- **无监督学习 (Unsupervised Learning)**：训练数据仅有输入特征 X 而无标签，目标是发现数据的内在结构，如聚类、降维、密度估计。
- **强化学习 (Reinforcement Learning)**：智能体 (Agent) 在环境 (Environment) 中通过试错、接收奖励信号来学习最优策略，典型场景如游戏 AI、机器人控制。
- **泛化能力 (Generalization)**：模型在训练集之外的新数据上的表现能力，是机器学习最核心的追求目标。
- **过拟合 (Overfitting)**：模型过度记忆训练数据中的噪声和细节，导致训练误差很低但测试误差很高。
- **欠拟合 (Underfitting)**：模型过于简单，无法捕捉数据中的基本规律，导致训练误差和测试误差都较高。
- **偏差-方差权衡 (Bias-Variance Tradeoff)**：偏差衡量模型预测期望值与真实值的偏离程度，方差衡量模型对不同训练集的预测波动程度。模型复杂度增加时偏差下降、方差上升，最优模型在二者之间取得平衡。

## 知识正文
机器学习是现代人工智能的核心支柱。与传统编程中程序员显式编写规则不同，机器学习让计算机自动从数据中归纳模式。这一范式的转变使得许多传统方法无法解决的复杂问题（如图像识别、自然语言理解）成为可能。

机器学习系统的构建通常遵循以下流程：首先收集和预处理数据，包括数据清洗、特征工程和标准化；然后将数据划分为训练集、验证集和测试集，通常比例为 6:2:2 或 8:1:1；接着选择适当的模型和算法进行训练；之后在验证集上评估和调参；最后在测试集上评估最终模型的泛化性能。

监督学习是最成熟、应用最广泛的学习范式。在分类任务中，常见算法包括逻辑回归、决策树、支持向量机和深度神经网络；在回归任务中，线性回归、岭回归和随机森林回归是常用选择。监督学习的评价指标包括准确率、精确率、召回率、F1 分数（分类）以及均方误差、平均绝对误差（回归）。

无监督学习在数据没有标签时发挥作用。K-Means 聚类将数据点分为 K 个簇，使簇内相似度高、簇间相似度低；层次聚类通过自底向上或自顶向下的方式构建数据的树状结构；DBSCAN 基于密度进行聚类，能发现任意形状的簇并排除噪声点。降维技术如 PCA 和 t-SNE 则用于高维数据的可视化和特征压缩。

强化学习的独特之处在于其交互性和序贯决策特性。智能体在每一步选择动作，环境反馈下一个状态和即时奖励，目标是最大化累积奖励。马尔可夫决策过程 (MDP) 是其数学框架，Q-Learning 和策略梯度是两类主要算法。AlphaGo 的成功是深度强化学习的里程碑式应用。

理解过拟合与欠拟合是机器学习的核心素养。当模型容量不足时出现欠拟合，表现为训练误差和测试误差都高；当模型容量过大或数据过少时出现过拟合，表现为训练误差极低但测试误差高。解决方案包括：对于欠拟合，增加模型复杂度或特征数量；对于过拟合，增加训练数据、使用正则化（L1/L2）、Dropout、早停或集成学习。

偏差-方差分解是理解泛化性能的理论框架。高偏差对应欠拟合，高方差对应过拟合。模型的期望泛化误差可分解为偏差平方、方差和不可约误差之和。最优模型需要在偏差和方差之间折中，这通常通过交叉验证来选择模型的超参数实现。

## 关键公式
- **均方误差 (MSE)**：$$\text{MSE} = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$，其中 $y_i$ 为真实值，$\hat{y}_i$ 为预测值，$n$ 为样本数。
- **偏差-方差分解**：$$\mathbb{E}[(y - \hat{f}(x))^2] = (\text{Bias}[\hat{f}(x)])^2 + \text{Var}[\hat{f}(x)] + \sigma^2$$，其中 $\sigma^2$ 为不可约误差。
- **准确率**：$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$，TP 为真正例，TN 为真负例，FP 为假正例，FN 为假负例。
- **正则化损失函数**：$$J(\theta) = L(\theta) + \lambda R(\theta)$$，其中 $L(\theta)$ 为原始损失函数，$R(\theta)$ 为正则化项，$\lambda$ 为正则化强度超参数。

## 示例
**示例 1：房价预测（回归）**  
假设我们有 1000 条房屋数据，每条包含面积、卧室数、地段等特征 X 和实际成交价 Y。我们将其划分为 800 条训练、100 条验证、100 条测试。使用线性回归在训练集上拟合，在测试集上得到 MAE = 15 万元。若训练集 MAE = 3 万元、测试集 MAE = 45 万元，则发生了严重的过拟合，应考虑正则化或减少特征。

**示例 2：垃圾邮件分类（二分类）**  
给定 5000 封已标注的邮件（垃圾/正常），使用逻辑回归或朴素贝叶斯训练分类器。若模型在训练集上准确率 99%、测试集上 82%，则存在过拟合。解决方案：增加数据量、使用更简单的模型或添加 L2 正则化。

## 常见误区
1. **"训练准确率高就是好模型"**：错误。高训练准确率可能只是过拟合的表象，真正衡量模型好坏的是测试集或验证集上的性能。
2. **"机器学习不需要人工干预"**：错误。特征工程、超参数调优、数据清洗等环节都需要大量的人工专业知识和经验。
3. **"数据越多模型越好"**：不完全正确。低质量、有偏的数据越多反而可能引入更多噪声和偏见。数据的质量和多样性比数量更重要。
4. **"深度学习可以解决所有问题"**：错误。深度学习需要大量数据和计算资源，在小样本、需要可解释性的场景中，传统机器学习方法往往更优。
5. **"验证集和测试集可以混用"**：错误。测试集只能在最终评估时使用一次，若反复在测试集上调参等同于数据泄露，会导致过于乐观的泛化估计。

## 基础练习
**题目 1（单选）**：以下哪种任务不属于监督学习？
A. 房价预测
B. 手写数字识别
C. 客户分群
D. 垃圾邮件检测
**正确答案**：C
**解析**：客户分群通常没有预先定义的标签，属于无监督学习中的聚类任务。A、B、D 分别对应回归、多分类和二分类，均有标签，属于监督学习。

**题目 2（单选）**：当模型训练误差很低而测试误差很高时，最可能发生了：
A. 欠拟合
B. 过拟合
C. 数据泄露
D. 特征缺失
**正确答案**：B
**解析**：训练误差低、测试误差高是过拟合的典型表现，说明模型过度记忆了训练数据中的噪声。

## 进阶练习
**题目**：假设你正在开发一个电影推荐系统。用户数据包含观影历史和评分，但你没有任何显式的"喜欢/不喜欢"标签。请说明：(1) 该问题属于哪种学习类型？(2) 可以如何构造训练信号？(3) 存在哪些潜在的评估挑战？

**参考答案**：
(1) 该问题可以建模为多种方式。若仅根据观影历史做用户聚类或电影聚类，属于无监督学习；若将评分作为标签预测，则属于监督学习（回归）；若将推荐视为序贯决策（推荐→用户反馈→调整推荐），则可用强化学习建模。

(2) 训练信号可从隐式反馈（点击、观看时长、停留时间）或显式反馈（评分、点赞、收藏）中构造。例如将评分 ≥ 4 视为正样本，评分 ≤ 2 视为负样本。

(3) 主要挑战包括：离线评估指标（如 RMSE）与在线业务指标（如点击率、留存率）之间存在 gap；用户偏好随时间漂移（概念漂移）；冷启动问题（新用户或新物品缺乏数据）；选择偏差（只能观察到系统推荐的物品的用户反馈，未推荐的物品存在反馈缺失）。
