# 模型评估与过拟合

## 学习目标
1. 掌握交叉验证的原理、常见变体及其在模型选择中的作用
2. 理解混淆矩阵及其派生指标（精确率、召回率、F1 分数）的计算与含义
3. 掌握 ROC 曲线和 AUC 的概念、绘制方法及解读
4. 理解超参数调优的常用策略（网格搜索、随机搜索、贝叶斯优化）
5. 掌握正则化技术（L1、L2、Dropout、早停）防止过拟合的原理

## 核心概念
- **交叉验证 (Cross-Validation)**：将数据集划分为 K 个等大互斥子集，每次用 K-1 个子集训练、余下 1 个验证，重复 K 次取平均。K 折交叉验证是评估模型泛化性能的黄金标准。
- **混淆矩阵 (Confusion Matrix)**：分类问题的核心评估工具，以矩阵形式记录 TP（真正例）、FP（假正例）、FN（假负例）、TN（真负例）的数量。
- **精确率与召回率 (Precision & Recall)**：精确率 = TP/(TP+FP)，衡量预测为正的样本中有多少确实为正；召回率 = TP/(TP+FN)，衡量所有真实正样本中有多少被正确识别。二者构成 tradeoff。
- **F1 分数**：$F1 = 2 \cdot \frac{P \cdot R}{P + R} = \frac{2TP}{2TP + FP + FN}$，精确率和召回率的调和平均。相比算术平均，调和平均对低值更敏感，只有当两者都高时 F1 才高。
- **ROC 与 AUC**：ROC 曲线以真正例率 (TPR = 召回率) 为纵轴、假正例率 (FPR = FP/(FP+TN)) 为横轴，绘制不同分类阈值下的性能。AUC 为曲线下面积，0.5 为随机分类器，1.0 为完美分类器。
- **L1 正则化 (Lasso)**：在损失函数中加入权重绝对值之和 $\lambda\sum|w_j|$，诱导稀疏权重向量，天然实现特征选择。
- **L2 正则化 (Ridge / 权重衰减)**：加入权重平方和 $\lambda\sum w_j^2$，使权重整体变小但不为零，防止任何单个特征对预测产生过大影响。
- **Dropout**：训练时以概率 p 随机"丢弃"（置零）隐藏层神经元的输出，每次前向传播等效于在采样子网络上训练。测试时使用完整网络但缩放权重。深度学习中极有效的正则化手段。

## 知识正文
模型评估是机器学习工作流中最容易被低估但又至关重要的环节。一个在训练集上表现完美的模型可能在实际部署中毫无价值——如果没有恰当的评估方法，我们无从知晓模型真实的泛化能力。

交叉验证是模型评估的基石方法。最简单的留出法 (Hold-out) 将数据一次性划分为训练集和测试集，简单但方差大、数据利用不充分。K 折交叉验证将数据均匀分为 K 份，轮流将每份作为验证集、其余作为训练集，K 次结果的平均作为模型性能的最终估计。这减少了评估方差，同时充分利用了所有数据。K 通常取 5 或 10。留一法 (LOO-CV) 是 K=N 的特例，评估很准但计算代价极高。对时序数据需使用时序交叉验证（按时间顺序滚动划分），避免"用未来预测过去"的数据泄露。分层 K 折 (Stratified K-Fold) 在每折中保持类别比例，强烈推荐用于分类问题。

混淆矩阵为分类模型提供了细粒度的错误分析视角。从中可衍生四个关键率：真正例率 TPR（越高越好）、假正例率 FPR（越低越好）、真负例率 TNR 和假负例率 FNR。但准确率在类别不平衡时严重失真——假设 99% 样本为负类，全预测为负就得 99% 准确率，但正类全部漏检。精确率和召回率专门应对这种场景：精确率关注"预测为正中有多少是对的"，召回率关注"真实正中有多少被找到了"。当二者不可兼得时，F1 作为调和平均更适合评估。根据具体业务需求，有时会有侧重——如癌症筛查优先保证召回率（不漏诊），垃圾邮件过滤优先把握精确率（不误删正常邮件）。

ROC 曲线是理解分类器在不同阈值下表现的有力工具。随着分类阈值从 0 降到 1（概率阈值从严格到宽松），TPR 和 FPR 单调不减。完美分类器 ROC 沿左边界和上边界，AUC=1；随机分类器 ROC 为对角线，AUC≈0.5。AUC 的一个优雅解释是：随机选一个正例和一个负例，分类器对正例打分高于负例的概率。这使得 AUC 不依赖于具体的分类阈值，是比准确率更全面的分类器评价指标。

超参数调优是模型开发中的迭代核心。网格搜索 (Grid Search) 穷举预设的超参数组合，概念直观但维度灾难下组合数爆炸。随机搜索 (Random Search, Bergstra & Bengio 2012) 在超参数空间中随机采样，往往在更少尝试中达到相近甚至更好的效果，因为只有少数超参数真正重要。贝叶斯优化基于高斯过程或树形模型构建代理模型来建模超参数→性能的映射，智能选择下一个最有价值的评估点，在评估代价高昂（如训练大型网络）时尤为有效。

正则化是防止过拟合的技术工具箱。L1 正则化产生稀疏解，相当于对参数施加拉普拉斯先验，从几何角度是因为菱形约束区域的角点容易成为最优解，那些角点对应某些特征权重恰为零；L2 正则化使权重缩小，对应高斯先验，圆形约束区域使最优解通常是各维度均非零但都较小。Dropout 可看作一种极端的集成方法——训练时每次随机丢弃部分神经元，等效于在指数多个稀疏子网络上训练并隐式集成，显著降低过拟合。早停 (Early Stopping) 则监控验证集误差，在误差不再下降时停止训练，是深度学习中最简单有效的正则化手段之一——它防止网络过度记忆训练数据的细节噪声。

## 关键公式
- **精确率**：$$Precision = \frac{TP}{TP + FP}$$
- **召回率**：$$Recall = \frac{TP}{TP + FN}$$
- **F1 分数**：$$F1 = 2 \cdot \frac{Precision \cdot Recall}{Precision + Recall} = \frac{2TP}{2TP + FP + FN}$$
- **L1 正则化损失**：$$J_{L1}(\mathbf{w}) = L(\mathbf{w}) + \lambda \sum_{j=1}^{d} |w_j|$$
- **L2 正则化损失**：$$J_{L2}(\mathbf{w}) = L(\mathbf{w}) + \lambda \sum_{j=1}^{d} w_j^2$$

## 示例
**示例 1：垃圾邮件过滤器的混淆矩阵**  
测试 1000 封邮件：实际 100 封垃圾邮件、900 封正常。分类器预测 80 封为垃圾（其中 70 封确实垃圾、10 封误判），剩余 920 封预测为正常（其中 30 封垃圾被漏掉）。

- TP=70, FP=10, FN=30, TN=890
- 准确率 = (70+890)/1000 = 96%
- 精确率 = 70/(70+10) = 87.5%
- 召回率 = 70/(70+30) = 70%
- F1 = 2 × (0.875 × 0.7)/(0.875 + 0.7) = 0.778

分析：96% 的准确率看起来很漂亮，但召回率仅 70%——仍有 30% 的垃圾邮件漏网。精确率 87.5% 意味着有 12.5% 的正常邮件被误判为垃圾。根据业务场景（用户更讨厌误删正常邮件），应调整阈值优先保证精确率。

**示例 2：5 折交叉验证选模型**  
对逻辑回归（不同 C 值）和 SVM（不同 γ）在 1000 条数据上做 5 折交叉验证。将数据均分 5 份，每轮用 800 训练、200 验证。对每个 C/γ 组合得 5 个验证集 AUC，取均值。最终选择均值 AUC 最高的组合，用全部 1000 条数据重新训练得到最终模型。整个过程在独立的测试集上报告最终性能。

## 常见误区
1. **"交叉验证结果就是模型上线后的性能"**：错误。交叉验证评估的是模型在当前数据分布下的性能，而线上数据可能随时间发生分布漂移（概念漂移）。交叉验证是必要而非充分的性能保证。
2. **"AUC 高模型就一定好"**：不全面。AUC 衡量的是整体排序质量，但在极度不平衡的数据中，AUC 可能高而精确率-召回率曲线（PR 曲线）却暴露问题。当正类极少时，PR 曲线比 ROC 曲线更具诊断价值。
3. **"更多正则化总是更好"**：错误。过强的正则化会导致欠拟合——模型因惩罚过重而无法学习数据中的真实模式。正则化强度 λ 需要通过交叉验证来选择合适的值。
4. **"Dropout 在推理时也应该启用"**：错误。Dropout 只在训练时使用以注入噪声和实现隐式集成。推理时关闭 Dropout，但需要对权重乘以保留概率 (1-p) 做缩放（或训练时做反向的 Inverted Dropout 缩放以避免推理时的额外操作）。
5. **"特征越多模型越好，让 Lasso 去挑特征就行"**：危险。Lasso 确实能做特征选择，但若特征数远大于样本数（高维小样本，如基因数据），Lasso 最多选 n 个特征（n 为样本数），且相关性高的特征组中 Lasso 可能随机选一个，丢失信息。此时弹性网络 (Elastic Net) 更稳健。

## 基础练习
**题目 1（单选）**：在极度不平衡的二分类问题中（正类仅占 1%），以下哪个指标最不适合单独用于评估模型？
A. AUC
B. F1 分数
C. 准确率
D. 精确率-召回率曲线
**正确答案**：C
**解析**：准确率在类别极不平衡时会严重失真。全部预测为负类可得 99% 准确率，但正类全部漏检，毫无实用价值。F1、AUC 和 PR 曲线都考虑了正负类的区分能力和平衡性。

**题目 2（单选）**：关于 Dropout 的描述，正确的是：
A. Dropout 在训练和推理时都以相同概率丢弃神经元
B. Dropout 可被视为一种隐式的模型集成方法
C. Dropout 丢弃的是输入特征而非隐藏层神经元
D. Dropout 的参数 p 越大表示丢弃的概率越大
**正确答案**：B
**解析**：Dropout 每次训练时随机丢弃不同的神经元子集，等效于在不同的稀疏子网络上训练，测试时相当于这些子网络的集成，因此是一种隐式集成。A 错误——推理时不丢弃；C 错误——丢弃的是神经元输出；D 通常 p 表示保留概率（或丢弃概率取决于框架约定，需查阅文档）。

## 进阶练习
**题目**：请解释为什么准确率在类别不平衡数据中不是合格的评估指标，并说明在以下两个场景中应优先选择哪个评估指标：(1) 癌症早期筛查，漏诊后果严重；(2) 搜索引擎的搜索结果排序。此外，分析为什么 ROC 曲线对类别不平衡不敏感而 PR 曲线敏感。

**参考答案**：
准确率失效的原因：假设负类占比 99%，分类器只需始终输出"负类"即可达 99% 准确率，但这种"模型"实际毫无分类能力。准确率将正负类错误视为同等重要，而现实中的错误代价往往不对称。

场景分析：
(1) 癌症筛查：优先关注召回率。漏诊（FN）意味着让患者错误地以为自己健康，可能错过最佳治疗时机。可接受一定比例的假阳性（FP，进一步做精确检查即可排除），也就是在保证高召回率的前提下尽量提高精确率。
(2) 搜索引擎排序：优先关注前 K 个结果的精确率（Precision@K），同时可用 NDCG 等考虑排序位置的指标。用户只看前几条结果，排在后面的相关性影响很小。

ROC vs PR 对不平衡的敏感性：ROC 的横轴 FPR = FP/(FP+TN)，当负类（TN）数量极大时，FP 的变动被巨大的 TN 分母稀释，FPR 变化很小，ROC 曲线看起来很"稳定"甚至"优秀"。PR 曲线以精确率（受 FP 直接影响）为纵轴，不涉及 TN。当正类极少时，少量 FP 就会显著拉低精确率，使 PR 曲线忠实地暴露出模型在少数类上的糟糕表现。因此在正类稀疏的场景中，PR 曲线比 ROC 曲线更有信息量。
