# 决策树

## 学习目标
1. 理解决策树的基本结构与递归划分思想
2. 掌握 ID3（信息增益）、C4.5（增益率）和 CART（基尼系数）三种分裂准则
3. 理解预剪枝与后剪枝的原理、实现方式及对泛化性能的影响
4. 掌握随机森林的 Bagging 集成思想和随机特征选择机制
5. 能够分析决策树在实际应用中的优缺点并选择合适的场景使用

## 核心概念
- **决策树 (Decision Tree)**：一种树形结构的分类与回归模型，通过递归地将数据按特征取值划分为更纯的子集来实现预测。内部节点代表特征测试，分支代表测试结果，叶节点代表预测输出。
- **信息熵 (Entropy)**：$H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k$，衡量数据集纯度的指标。熵越大，数据越混杂；熵为 0 时所有样本属于同一类。
- **信息增益 (Information Gain)**：$IG(D, A) = H(D) - \sum_v \frac{|D_v|}{|D|} H(D_v)$，按特征 A 划分后熵的减少量。ID3 算法以此选择最优划分特征。
- **增益率 (Gain Ratio)**：$GR(D, A) = \frac{IG(D, A)}{H_A(D)}$，在信息增益基础上除以特征 A 的"固有值"以避免对可取值多的特征偏好的问题。C4.5 先选信息增益高于均值的特征，再从中选增益率最高的。
- **基尼系数 (Gini Index)**：$Gini(D) = 1 - \sum_k p_k^2$，衡量不纯度。CART 分类树使用基尼系数作为分裂准则，计算效率高于熵。
- **剪枝 (Pruning)**：通过减少树的节点数来防止过拟合。预剪枝在构建过程中提前停止分裂，后剪枝在完全生长后自底向上合并或删除节点。
- **随机森林 (Random Forest)**：Bagging 集成多棵决策树，每棵树在随机抽样的样本子集和随机选择的特征子集上训练，最后通过投票（分类）或平均（回归）汇总预测结果。
- **CART (Classification and Regression Tree)**：同时支持分类和回归的二叉树生成算法。分类用基尼系数，回归用 MSE 作为分裂准则，且强制每个分支都是二叉树。

## 知识正文
决策树是一种直观且强大的机器学习模型。其本质是递归地将数据空间划分为不重叠的矩形区域，并在每个区域内用简单模型（如多数类或均值）做预测。这种"分而治之"的思想与人类的逻辑推理方式高度吻合，使得决策树成为可解释性最强的黑箱模型之一。

决策树的构建过程是从根节点开始，选择最优特征和划分点，将当前节点数据集分裂为若干子集，再对每个子节点递归重复此过程，直到满足停止条件。关键在于如何定义"最优划分"。ID3 算法（Quinlan, 1986）引入信息增益作为标准：选择使划分后信息熵下降最多的特征。但信息增益对可取值较多的特征有天然偏好——一个特征若取值极多（如 ID 号），每个取值只对应一个样本，划分后熵为 0，信息增益最大，但毫无泛化能力。

C4.5 通过增益率修正了上述偏差：在信息增益的基础上除以特征自身的"分裂信息值"，惩罚取值过多的特征。同时 C4.5 支持连续值离散化（二分法）、缺失值处理和剪枝等功能，成为最广泛使用的决策树算法之一。

CART 算法（Breiman, 1984）是另一种主流方法，它与 ID3/C4.5 的主要区别在于：生成的是二叉树而非多叉树；分类时使用基尼系数而非信息增益；回归时使用 MSE 最小化作为分裂准则。基尼系数可视为熵的泰勒近似，计算更简洁，且在实际效果上与信息增益相差无几。

未剪枝的决策树极易过拟合——树可以一直生长到每个叶节点只包含一个样本，完美拟合训练数据但在测试集上表现糟糕。剪枝是解决此问题的核心技术。预剪枝在分裂前评估：若当前划分不能显著提升验证集精度则停止。后剪枝则先让树充分生长，再从叶向根评估每个非叶节点：若将其替换为叶节点或最常用分支能提升验证集精度则执行剪枝。后剪枝通常效果更好，但计算开销更大。

随机森林是决策树的最著名集成变体。它通过两个层次的随机化来增加基学习器（决策树）之间的多样性：Bootstrap 抽样（从原数据中有放回地抽取与原始样本等大的训练集）和随机子空间（每次分裂时只从随机选取的 $\sqrt{d}$ 或 $\log d$ 个特征中选择最优划分）。两大随机化使各决策树产生不同的错误模式，集成后显著降低方差而不过多增加偏差，在实践中往往是性能最好的"开箱即用"算法之一。

决策树的优势在于：不需要特征标准化、能自然处理类别特征和缺失值、模型可解释性强、训练预测速度快。劣势包括：单棵树容易过拟合、对数据微小变化敏感（高方差）、对线性关系的拟合效率低于线性模型、在处理不平衡数据时容易偏向多数类。

## 关键公式
- **信息熵**：$$H(D) = -\sum_{k=1}^{K} \frac{|C_k|}{|D|} \log_2 \frac{|C_k|}{|D|}$$，其中 $C_k$ 为第 k 类的样本集合。
- **信息增益**：$$IG(D, A) = H(D) - \sum_{v \in \text{Values}(A)} \frac{|D_v|}{|D|} H(D_v)$$。
- **增益率**：$$GR(D, A) = \frac{IG(D, A)}{H_A(D)}$$，$$H_A(D) = -\sum_v \frac{|D_v|}{|D|} \log_2 \frac{|D_v|}{|D|}$$。
- **基尼系数**：$$Gini(D) = 1 - \sum_{k=1}^{K} \left(\frac{|C_k|}{|D|}\right)^2$$，取值范围 $[0, \frac{K-1}{K}]$。
- **CART 分类划分后基尼值**：$$Gini(D, A) = \sum_{v} \frac{|D_v|}{|D|} Gini(D_v)$$。

## 示例
**示例 1：是否打网球决策树**  
经典数据集：根据天气（晴/阴/雨）、温度（热/温/凉）、湿度（高/正常）、风力（强/弱）决定是否打网球。14 条数据，正例（打）9 条，负例（不打）5 条。

根节点熵：$H = -\frac{9}{14}\log_2\frac{9}{14} - \frac{5}{14}\log_2\frac{5}{14} \approx 0.940$。

按天气划分：晴（5 条，2+ 3-，熵 0.971）、阴（4 条，4+ 0-，熵 0）、雨（5 条，3+ 2-，熵 0.971）。加权熵 = $\frac{5}{14}\times 0.971 + \frac{4}{14}\times 0 + \frac{5}{14}\times 0.971 = 0.694$。信息增益 = 0.940 - 0.694 = 0.246，为所有特征中最大，故选天气为根节点。阴天分支已纯（全为正例），直接置为叶节点。

**示例 2：随机森林预测**  
100 棵树组成的随机森林对某样本分类。55 棵树预测 A 类、45 棵预测 B 类，最终投票结果为 A 类。若需要概率估计，则 A 类概率为 0.55。

## 常见误区
1. **"信息增益越大的特征越好"**：不完全正确。信息增益偏向于可取值多的特征（如编号 ID），增益率为更合理的选择标准。
2. **"决策树的深度越大越好"**：错误。深度过大会导致严重过拟合，树的复杂度需通过剪枝或超参数（max_depth, min_samples_split）来控制。
3. **"随机森林中的每棵树都与普通决策树一样"**：不准确。随机森林中的每棵树只使用 Bootstrap 抽样数据和随机特征子集，比单棵决策树更弱（高偏差），但集成后通过多样性抵消，整体偏差略增、方差大幅下降。
4. **"决策树不能处理缺失值"**：不正确。C4.5 通过概率权重分配处理缺失值：样本以权重方式同时进入所有分支，权重按各分支样本比例分配。
5. **"CART 只能做分类"**：错误。CART 的全称就是"分类与回归树"，回归树使用 MSE 最小化选择分裂点，叶节点输出为该区域目标值的均值。

## 基础练习
**题目 1（单选）**：C4.5 算法使用增益率而非信息增益选择特征的主要原因是：
A. 增益率计算更快
B. 信息增益偏向于可取值较多的特征
C. 增益率能处理连续值
D. 信息增益在分类问题中不适用
**正确答案**：B
**解析**：信息增益对可取值较多的特征有天然偏好（极端的例子是唯一 ID），因为这些特征划分后容易得到纯的子集。增益率通过除以特征的"固有值"来惩罚这种偏向。

**题目 2（单选）**：关于预剪枝和后剪枝，以下说法正确的是：
A. 预剪枝总是比后剪枝效果好
B. 后剪枝比预剪枝计算开销小
C. 预剪枝可能因"目光短浅"导致欠拟合
D. 后剪枝不需要验证集
**正确答案**：C
**解析**：预剪枝基于当前划分的即时收益做决定，可能阻止了后续有增益的组合划分（如当前划分无收益但再划分子节点后有收益），导致欠拟合。后剪枝先让树充分生长再回溯剪枝，通常泛化性能更好，但计算开销更大。两者都需要验证集来评估剪枝效果。

## 进阶练习
**题目**：分析随机森林为什么通常比单棵决策树泛化性能更好，从偏差-方差分解的角度给出解释。并说明随机森林的两个核心随机化各自的作用。

**参考答案**：
从偏差-方差分解角度，单棵决策树对训练数据的微小变化非常敏感（高方差），容易过拟合。随机森林通过 Bagging（Bootstrap Aggregating）平均多棵树的结果，根据 $Var(\bar{X}) = Var(X)/n$（独立同分布假设）或更一般地 $Var(\bar{X}) = \rho\sigma^2 + \frac{1-\rho}{n}\sigma^2$（存在相关性 $\rho$ 时），集成显著降低方差。

两个核心随机化：
1. Bootstrap 抽样：每棵树训练在不同的 Bootstrap 样本上，增加了树之间的多样性。由于是有放回抽样，约 36.8% 的样本（Out-Of-Bag）被排除在外，可作为免费的验证集估计泛化误差。
2. 随机子空间：每次分裂时只从总特征中随机选 $\sqrt{d}$（分类）或 $d/3$（回归）个特征作为候选，去除了强特征对每次分裂的垄断，使各树关注不同特征组合，进一步降低树间的相关性 $\rho$。

相关性 $\rho$ 越低，方差下降越显著。两块随机化分别从样本和特征维度注入多样性，使得即使基学习器（决策树）本身方差大，集成的整体方差也得到有效控制，同时偏差仅略微增加。这解释了随机森林在各种任务中稳定优异的表现。
