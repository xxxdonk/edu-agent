# EduAgent 最终演示脚本

建议主视频时长：6 分 30 秒以内

演示课程：《机器学习基础》

固定案例数据：`scripts/demo_cases.json`

## 1. 演示前检查

1. 在根目录运行模型预热：

   ```powershell
   .\.venv\Scripts\python.exe .\scripts\verify_llm_profile_planner.py
   ```

2. 一键启动：

   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\start_demo.ps1
   ```

3. 确认：

   - `/api/health` 返回服务和数据库均为 `ok`；
   - 页面健康状态正常；
   - Profile 和 Planner 显示结构化大模型结果；
   - 使用新的学生 ID，避免把旧任务误当成新演示；
   - 浏览器缩放 100%，窗口至少 1366×768；
   - 关闭通知和自动更新弹窗。

4. 录制前运行：

   ```powershell
   .\.venv\Scripts\python.exe .\scripts\verify_end_to_end.py --verify-cache
   .\.venv\Scripts\python.exe .\scripts\verify_demo_cases.py
   ```

   三个案例脚本默认每个案例最多透明尝试 4 次，每次使用新的学生 ID；输出中的 `attempt_count` 和 `prior_failure_codes` 必须保留，以便演示人员看到真实模型波动。产品 Agent 的单次格式修复上限不因此改变。

   脚本只输出安全摘要。不要在画面中打开 `.env`、终端环境变量或完整模型响应。

## 2. 主视频流程

主视频建议使用案例 C“项目实践型学生”，它最容易完整展示画像、代码资源、Quiz 和动态重规划。

### 0:00–0:30 开场

画面：EduAgent 学习工作台。

讲解：

“EduAgent 是由动态学习画像驱动的多智能体个性化学习系统。学生只需描述自己的情况，系统就能完成画像、路径、课程检索、五类资源、审校、练习评价和动态调整。”

指出顶部完整流程和后端健康状态。

### 0:30–1:25 自然语言画像

输入：

> 我是人工智能专业大二学生，目前课程是机器学习，已有Python和机器学习基础，正在做一个客户流失分类项目，对逻辑回归、模型选择和参数调优不熟。希望完成可运行项目并比较模型效果，每天可以学习90分钟，偏好代码案例、实验和项目任务。

讲解：

“Profile Agent 使用截至本轮的完整对话历史，生成带字段级证据、来源和置信度的画像。这里可以看到专业、课程、已有基础、项目目标、薄弱点、资源偏好和时间预算。”

操作：

- 展开逻辑回归或模型选择的证据；
- 指出画像版本；
- 指出 `llm_structured` 模式。

若页面显示 development 模式，应明确说明这是降级结果，不将其描述为 DeepSeek 结构化结果。

### 1:25–2:10 个性化路径

讲解：

“Planner 使用同一画像版本安排有依赖关系的步骤。项目型学生应先补逻辑回归、模型选择或调参，再进入客户流失分类项目，而不是得到固定课程目录。”

操作：

- 展示 `topic`、`learning_goal`、`reason`；
- 展示完成标准、预计时间和前置知识；
- 选择最适合生成资源的当前步骤。

### 2:10–3:25 多智能体生成与 SSE

点击“生成五类资源”。

讲解：

“Orchestrator 真正并行调度五个资源 Agent。每类资源都先从课程知识库检索，再由 Reviewer 单独检查。页面通过可恢复 SSE 持续接收 started、completed 或 failed 事件。单项失败不会取消其他资源。”

操作：

- 指出 Retriever、五个 Agent 和 Reviewer；
- 指出 sequence 连续；
- 指出任务最终为 `completed` 或如实说明 `partial_success`；
- 如果出现 `cache_hit=true`，说明复用的是相同画像、路径、模型和知识库版本下的已审校快照，命中后仍会重新审校。

为了展示真实冷生成，可使用新学生或 `regenerate=true`。不要把缓存命中时间描述为模型重新生成耗时。

### 3:25–4:40 五类资源

依次展示：

1. Markdown 讲解；
2. Mermaid 思维导图；
3. 基础、进阶、挑战三层 Quiz；
4. 拓展阅读；
5. Python 项目代码。

每张卡片指出：

- 标题和类型；
- 目标知识点；
- 难度；
- 与项目目标、薄弱点或偏好的个性化原因；
- 真实课程来源；
- Reviewer 状态。

讲解：

“资源的公共结构完全一致，但内容依据画像和当前路径步骤变化。来源来自本地课程知识库，没有来源的内容不会被伪装为成功资源。”

### 4:40–5:40 Quiz 与 Evaluation

按案例 C 的提交策略：基础题答对，其余题根据演示目的保留错误或不完整回答。

讲解：

“Evaluation 使用已持久化 Quiz 的真实答案评分，不按回答长度猜测正确性。它识别薄弱知识点，并把证据明确标记为 evaluation。”

操作：

- 展示掌握度和是否通过；
- 展示薄弱知识点和反馈；
- 展示答案解析。

### 5:40–6:20 动态闭环

讲解：

“Evaluation 在后端同一次请求中创建画像新版本，再让 Planner 根据旧路径和评价摘要重新规划。评价不会伪装成学生原话。”

操作：

- 展示画像 v1 → v2；
- 展示 evaluation 证据；
- 展示新路径；
- 指出 `adjustment_reason`。

### 6:20–6:30 收尾

讲解：

“EduAgent 的核心价值，是把一次推荐升级为由可追溯证据、真实课程来源和学习评价持续驱动的多智能体学习闭环。”

## 3. 三个固定演示案例

### A. 初学视觉型学生

输入：

> 我是计算机科学专业大一学生，刚开始学习机器学习，线性代数基础和梯度下降都很薄弱，希望看懂基础分类模型。每天可以学习30分钟，偏好图示、思维导图和生活类比。

预期画像：

- 专业包含计算机；
- 课程为机器学习；
- 知识水平为 beginner；
- 薄弱点包含线性代数或数学基础、梯度下降；
- 目标包含分类；
- 每天 30 分钟；
- 偏好思维导图、图示和视觉表达。

预期路径：

- 优先补线性代数、数学基础或梯度下降；
- 后续进入基础分类模型；
- 单步时间适合 30 分钟预算。

预期资源差异：

- 讲解先使用直观概念和生活类比；
- MindMap 强调概念关系；
- 公式和代码难度保持 beginner；
- 个性化原因引用视觉偏好和数学薄弱点。

Quiz 提交示例：

- 三题全部故意答错；
- 用时 8 分钟。

评价后预期：

- 画像新增 evaluation 证据；
- 数学基础和梯度下降继续作为优先薄弱点；
- 新路径先复习基础概念。

### B. 考试型学生

输入：

> 我是数据科学专业大三学生，正在复习机器学习期末考试，已经学过回归和分类，但模型评估、过拟合和正则化容易混淆。目标是考试达到85分，每天可以学习60分钟，偏好分层练习、公式总结和错题解析。

预期画像：

- 专业为数据科学；
- 课程为机器学习；
- 知识水平为 intermediate 或 advanced；
- 薄弱点包含模型评估、过拟合、正则化；
- 目标包含期末考试和 85 分；
- 每天 60 分钟；
- 偏好练习、公式总结和错题解析。

预期路径：

- 优先辨析模型评估、过拟合和正则化；
- 后续包含综合练习或考试复习；
- 完成标准体现题目正确率和概念辨析。

预期资源差异：

- Quiz 和讲解突出公式、易混概念与分层练习；
- Reading 更偏复习总结；
- 个性化原因引用考试目标。

Quiz 提交示例：

- 正误交替提交；
- 用时 12 分钟。

评价后预期：

- 错题知识点成为 evaluation 证据；
- 新路径增加针对性复习和练习完成标准。

### C. 项目实践型学生

输入：

> 我是人工智能专业大二学生，目前课程是机器学习，已有Python和机器学习基础，正在做一个客户流失分类项目，对逻辑回归、模型选择和参数调优不熟。希望完成可运行项目并比较模型效果，每天可以学习90分钟，偏好代码案例、实验和项目任务。

预期画像：

- 专业为人工智能；
- 课程为机器学习；
- 知识水平为 intermediate 或 advanced；
- 薄弱点包含逻辑回归、模型选择和参数调优；
- 目标包含项目和分类；
- 每天 90 分钟；
- 偏好代码、实验和实践。

预期路径：

- 优先逻辑回归、模型选择或调参；
- 后续进入分类项目和效果比较；
- 完成标准体现可运行实验与模型指标。

预期资源差异：

- Coding 围绕可运行分类实验、模型比较和调参；
- 讲解与 Quiz 服务于项目决策；
- 个性化原因引用客户流失分类项目。

Quiz 提交示例：

- 第一题正确，其余题用于暴露未掌握内容；
- 用时 15 分钟。

评价后预期：

- 项目相关薄弱点更新为 evaluation 证据；
- 新路径保留代码实践并优先安排未掌握的模型选择或调参内容。

## 4. 案例差异验证

运行：

```powershell
.\.venv\Scripts\python.exe .\scripts\verify_demo_cases.py
```

验证器为每个案例创建唯一 `student_id`，并检查：

- Profile 和 Planner 模式；
- 画像锚点、时间预算和偏好；
- 路径主题的案例特异性；
- 五类资源、来源和 Reviewer；
- Quiz 策略、Evaluation、画像版本和新路径；
- 三个案例的画像、路径与资源个性化摘要明显不同。

输出只包含模式、计数、状态、知识点摘要、脱敏指纹和耗时。

## 5. 故障应对

- 模型超时或格式修复失败：展示明确的 development fallback 标记，不声称为模型结果。
- 单资源失败：展示 `partial_success` 和其他成功资源。
- SSE 中断：等待页面按 sequence 续传并使用轮询兜底。
- Mermaid 渲染失败：保留原始 Mermaid 内容，继续展示其他资源。
- 第二次生成很快：先确认 `cache_hit=true`，如需冷生成则选择新学生或强制重新生成。
- Evaluation 拒绝提交：确认题目 ID 来自当前学生、路径和步骤的持久化 Quiz。
