# EduAgent 最终演示脚本

建议主视频时长：6 分 50 秒以内

演示课程：高中数学、大学英语与机器学习项目

固定案例数据：`scripts/demo_cases.json`

## 1. 演示前检查

1. 在根目录运行安全预检：

   ```powershell
   .\.venv\Scripts\python.exe .\scripts\preflight_demo.py
   ```

   预检不生成完整资源，只检查依赖、模型配置存在性、知识库、端口、公共契约和提交卫生，且不显示 API Key。

2. 一键启动：

   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\start_demo.ps1
   ```

3. 确认：

   - `/api/health` 返回服务和数据库均为 `ok`；
   - 页面健康状态正常；
   - Profile 和 Planner 显示结构化大模型结果；
   - 使用新的学生 ID，避免把旧任务误当成新演示；
   - 浏览器缩放 100%，窗口至少 1366×768；
   - 关闭通知和自动更新弹窗。

4. 录制前运行：

   ```powershell
   .\.venv\Scripts\python.exe .\scripts\verify_end_to_end.py --verify-cache
   .\.venv\Scripts\python.exe .\scripts\verify_demo_cases.py
   ```

   三个案例脚本默认每个案例最多透明尝试 4 次，每次使用新的学生 ID；输出中的 `attempt_count` 和 `prior_failure_codes` 必须保留，以便演示人员看到真实模型波动。产品 Agent 的单次格式修复上限不因此改变。

   脚本只输出安全摘要。不要在画面中打开 `.env`、终端环境变量或完整模型响应。

## 2. 主视频流程

主视频使用案例 C“项目实践型学生”集中展示完整闭环，并在开场快速展示案例 A 高中数学与案例 B 大学英语，证明同一系统可切换学科且不会复用旧画像。案例 A 也可作为无本地 RAG 命中时透明降级的备用主案例。

### 0:00–0:30 开场

画面：EduAgent 学习工作台。

讲解：

“EduAgent 是由动态学习画像驱动的多智能体个性化学习系统。学生只需描述自己的情况，系统就能完成画像、路径、课程检索、五类资源、审校、练习评价和动态调整。”

指出顶部完整流程和后端健康状态。

### 0:30–1:25 自然语言画像

输入：

> 我是人工智能专业大二学生，目前课程是机器学习，已有Python和机器学习基础，正在做一个客户流失分类项目，对逻辑回归、模型选择和参数调优不熟。希望完成可运行项目并比较模型效果，每天可以学习90分钟，偏好代码案例、实验和项目任务。

讲解：

“Profile Agent 使用截至本轮的完整对话历史，生成带字段级证据、来源和置信度的画像。这里可以看到专业、课程、考试目标、薄弱点、资源偏好和时间预算。”

操作：

- 展开逻辑回归或模型选择的证据；
- 指出画像版本；
- 指出 `llm_structured` 模式。

若页面显示 development 模式，应明确说明这是降级结果，不将其描述为 DeepSeek 结构化结果。

### 1:25–2:10 个性化路径

讲解：

“Planner 使用同一画像版本安排有依赖关系的步骤。项目型学生会优先补逻辑回归、模型选择和参数调优，再进入客户流失分类实验，而不是得到固定课程目录。”

操作：

- 展示 `topic`、`learning_goal`、`reason`；
- 展示完成标准、预计时间和前置知识；
- 选择最适合生成资源的当前步骤。

### 2:10–3:25 多智能体生成与 SSE

点击“生成五类资源”。

讲解：

“Orchestrator 真正并行调度五个资源 Agent。每类资源都先从课程知识库检索，再由 Reviewer 单独检查。页面通过可恢复 SSE 持续接收 started、completed 或 failed 事件。单项失败不会取消其他资源。”

操作：

- 指出 Retriever、五个 Agent 和 Reviewer；
- 指出 sequence 连续；
- 指出任务最终为 `completed` 或如实说明 `partial_success`；
- 如果出现 `cache_hit=true`，说明复用的是相同画像、路径、模型和知识库版本下的已审校快照，命中后仍会重新审校。

为了展示真实冷生成，可使用新学生或 `regenerate=true`。不要把缓存命中时间描述为模型重新生成耗时。

### 3:25–4:40 五类资源

依次展示：

1. 资源包总览与推荐学习顺序；
2. 12 段 Markdown 讲解及 FAQ；
3. Mermaid 思维导图；
4. 分层拓展阅读与术语表；
5. 可运行的客户流失 Python 实验；
6. 8 至 12 题的基础、进阶、挑战 Quiz。

每张卡片指出：

- 标题和类型；
- 目标知识点；
- 难度；
- 与考试目标、薄弱点或练习偏好的个性化原因；
- 本地课程来源或透明的通用生成标记；
- Reviewer 状态。

讲解：

“五类资源承担不同学习功能，但沿用同一公共 Resource 契约。页面明确展示项目目标、薄弱点、偏好和时间预算；本地知识库命中时展示真实资料，未命中时明确标记为通用生成，不伪造课程来源。”

点击一个“学习追问建议”，说明它只填入对话输入框、允许编辑且不会自动发送。

### 4:40–5:40 Quiz 与 Evaluation

按案例 C 的提交策略保留第一题正确，其余题选择或输入不完整答案，用时 15 分钟，以展示项目相关薄弱点进入 evaluation 证据。使用题号导航展示作答进度和答案保留。

讲解：

“Evaluation 使用已持久化 Quiz 的真实答案评分，不按回答长度猜测正确性。它识别薄弱知识点，并把证据明确标记为 evaluation。”

操作：

- 展示掌握度和是否通过；
- 逐题指出绿色“正确”和红色“回答不正确”状态；
- 展示薄弱知识点和反馈；
- 展示答案解析。

### 5:40–6:20 动态闭环

讲解：

“Evaluation 在后端同一次请求中创建画像新版本，再让 Planner 根据旧路径和评价摘要重新规划。评价不会伪装成学生原话。”

操作：

- 展示画像 v1 → v2；
- 展示 evaluation 证据；
- 并排展示路径更新前后变化；
- 指出 `adjustment_reason`。

### 6:20–6:30 收尾

讲解：

“EduAgent 的核心价值，是把一次推荐升级为由可追溯证据、透明内容来源和学习评价持续驱动的多智能体学习闭环。”

## 3. 三个固定演示案例

### A. 高中数学基础型学生

输入：

> 我是高二学生，正在学习高中数学，函数和数列基础较弱，目标是提高期末考试成绩。我每天能学习45分钟，喜欢图示、例题和分步骤讲解。

预期画像：

- 学习阶段为高中或高二；
- 课程为高中数学；
- 薄弱点包含函数和数列；
- 目标包含期末考试或成绩提升；
- 每天 45 分钟；
- 偏好思维导图、图示和视觉表达。

预期路径：

- 优先补函数、数列和概念基础；
- 后续进入公式、例题、分层练习与错题整理；
- 单步时间适合 45 分钟预算。

预期资源差异：

- 讲解包含定义、公式、例题、步骤与易错点；
- MindMap 强调概念、公式、题型和错题；
- coding 类型显示为“计算与实验实践”；
- 未命中本地资料时明确显示通用模型模式，不出现机器学习来源。

Quiz 提交示例：

- 9 题全部故意答错；
- 用时 8 分钟。

评价后预期：

- 画像新增 evaluation 证据；
- 函数和数列继续作为优先薄弱点；
- 新路径先复习基础概念。

### B. 大学英语考试型学生

输入：

> 我是大学生，准备大学英语考试，阅读理解和写作较弱，希望进行阶段复习。我每天能学习60分钟，喜欢词汇清单、阅读练习和错题分析。

预期画像：

- 学习阶段为大学；
- 课程为大学英语；
- 薄弱点包含阅读理解和写作；
- 目标包含英语考试和阶段复习；
- 每天 60 分钟；
- 偏好练习、公式总结和错题解析。

预期路径：

- 优先安排词汇基础、阅读理解和写作表达；
- 后续包含纠错、综合练习和考试复习；
- 完成标准体现阅读正确率和写作修改。

预期资源差异：

- Quiz 和讲解覆盖词汇、阅读、写作和表达；
- Reading 更偏文本分析与表达方法；
- coding 类型显示为“应用实践任务”，不强制 Python。

Quiz 提交示例：

- 正误交替提交；
- 用时 12 分钟。

评价后预期：

- 错题知识点成为 evaluation 证据；
- 新路径增加针对性复习和练习完成标准。

### C. 项目实践型学生

输入：

> 我是人工智能专业大二学生，目前课程是机器学习，已有Python和机器学习基础，正在做一个客户流失分类项目，对逻辑回归、模型选择和参数调优不熟。希望完成可运行项目并比较模型效果，每天可以学习90分钟，偏好代码案例、实验和项目任务。

预期画像：

- 专业为人工智能；
- 课程为机器学习；
- 知识水平为 intermediate 或 advanced；
- 薄弱点包含逻辑回归、模型选择和参数调优；
- 目标包含项目和分类；
- 每天 90 分钟；
- 偏好代码、实验和实践。

预期路径：

- 优先逻辑回归、模型选择或调参；
- 后续进入分类项目和效果比较；
- 完成标准体现可运行实验与模型指标。

预期资源差异：

- Coding 围绕可运行分类实验、模型比较和调参；
- 讲解与 Quiz 服务于项目决策；
- 个性化原因引用客户流失分类项目。

Quiz 提交示例：

- 第一题正确，其余题用于暴露未掌握内容；
- 用时 15 分钟。

评价后预期：

- 项目相关薄弱点更新为 evaluation 证据；
- 新路径保留代码实践并优先安排未掌握的模型选择或调参内容。

## 4. 案例差异验证

运行：

```powershell
.\.venv\Scripts\python.exe .\scripts\verify_demo_cases.py
```

验证器为每个案例创建唯一 `student_id`，并检查：

- Profile 和 Planner 模式；
- 画像锚点、时间预算和偏好；
- 路径主题的案例特异性；
- 五类资源、来源和 Reviewer；
- Quiz 策略、Evaluation、画像版本和新路径；
- 三个案例的画像、路径与资源个性化摘要明显不同。

输出只包含模式、计数、状态、知识点摘要、脱敏指纹和耗时。

## 5. 故障应对

- 模型超时或格式修复失败：展示明确的 development fallback 标记，不声称为模型结果。
- 单资源失败：展示 `partial_success` 和其他成功资源。
- SSE 中断：等待页面按 sequence 续传并使用轮询兜底。
- Mermaid 渲染失败：保留原始 Mermaid 内容，继续展示其他资源。
- 第二次生成很快：先确认 `cache_hit=true`，如需冷生成则选择新学生或强制重新生成。
- Evaluation 拒绝提交：确认题目 ID 来自当前学生、路径和步骤的持久化 Quiz。
